# 可视化网页构造器开发文档

## 1. 项目概述

### 1.1 项目名称
可视化网页构造器 (Visual Web Builder)

### 1.2 项目描述
一个基于AI的可视化网页构造工具，允许用户通过选择DOM元素并输入自然语言提示词，让AI生成相应的代码来修改网页内容和结构。

### 1.3 核心功能
- DOM元素选择：使用现有的选择器工具选择网页中的DOM元素
- 自然语言交互：通过提示词描述修改意图
- AI代码生成：将用户意图转换为可执行的HTML/CSS/JavaScript代码
- 实时预览：预览修改效果
- 历史记录：记录修改历史，支持撤销/重做
- 代码管理：查看和编辑生成的代码

## 2. 技术架构

### 2.1 技术栈
- **前端**：原生JavaScript、HTML5、CSS3
- **AI集成**：OpenAI API 或其他LLM API
- **现有组件**：SeeWeb选择器工具

### 2.2 系统架构

```
┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│   用户界面层        │────▶│   核心逻辑层        │◀────│   AI服务层          │
└─────────────────────┘     └─────────────────────┘     └─────────────────────┘
        ▲                        ▲                        ▲

                          └─────────────────────┘
                                 ▲
                                 │
                          ┌─────────────────────┐
                          │   DOM操作层        │
                          └─────────────────────┘
```

### 2.3 模块划分
1. **选择器模块**：基于现有的SeeWeb选择器工具，用于选择DOM元素
2. **AI交互模块**：处理与AI的通信，发送提示词并接收生成的代码
3. **代码执行模块**：执行AI生成的代码，修改DOM
4. **用户界面模块**：提供用户交互界面，包括提示词输入、预览等
5. **历史记录模块**：记录修改历史，支持撤销/重做
6. **配置模块**：管理API密钥、模型选择等配置

## 3. 核心功能设计

### 3.1 DOM元素选择
- **功能**：使用现有的SeeWeb选择器工具选择网页中的DOM元素
- **实现**：集成现有的choseDiv（单选）和choseRect（扩选）功能
- **交互**：用户点击选择器按钮进入选择模式，选择完成后返回编辑模式

### 3.2 自然语言交互
- **功能**：用户输入自然语言提示词，描述对选中元素的修改意图
- **实现**：提供文本输入框，支持多行输入和示例提示
- **示例**：
  - "将这个按钮改为蓝色，添加圆角"
  - "在这个区域添加一个产品卡片，包含图片、标题和价格"
  - "将这个列表改为网格布局"

### 3.3 AI代码生成
- **功能**：将用户意图转换为可执行的HTML/CSS/JavaScript代码
- **实现**：
  - 构建提示词模板，包含选中元素的当前代码和用户意图
  - 调用AI API生成代码
  - 解析和验证生成的代码
- **API集成**：使用OpenAI API或其他LLM API

### 3.4 实时预览
- **功能**：预览修改效果
- **实现**：
  - 在沙箱环境中执行生成的代码
  - 显示修改前后的对比
  - 提供确认/取消按钮

### 3.5 历史记录
- **功能**：记录修改历史，支持撤销/重做
- **实现**：
  - 保存每次修改的前后状态
  - 提供撤销/重做按钮
  - 显示修改历史列表

### 3.6 代码管理
- **功能**：查看和编辑生成的代码
- **实现**：
  - 提供代码编辑器，显示生成的代码
  - 允许用户手动编辑代码
  - 提供代码格式化和语法高亮

## 4. 用户界面设计

### 4.1 主界面布局
- **顶部工具栏**：包含选择器按钮、AI助手按钮、历史记录按钮等
- **右侧面板**：
  - 提示词输入区域
  - AI生成代码显示区域
  - 预览控制区域
- **底部状态栏**：显示当前选中元素信息、修改状态等

### 4.2 交互流程
1. 用户点击选择器按钮进入选择模式
2. 用户选择一个或多个DOM元素
3. 系统显示右侧面板，用户输入修改意图
4. 用户点击"生成代码"按钮
5. AI生成代码并显示在面板中
6. 用户预览效果，确认或取消修改
7. 系统应用修改并记录到历史记录

### 4.3 响应式设计
- 适配不同屏幕尺寸
- 在小屏幕设备上优化布局
- 支持触摸操作
